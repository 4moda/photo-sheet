# Product Decisions (Current)

## ツールバー / パネル

- フローティングバーは `見た目 / 調整 / タイトル / 書き出し`（上限 6 個。2026-07-17 に「調整」を追加）
- パネル外タップで閉じる
- 写真操作はタップメニュー + 長押しドラッグ並べ替え
- シート全体の操作（撮影順に並べ替え・すべて削除）は右上 [+] メニューに集約

## デフォルト値

- 列数: `6`
- 用紙: `8x10`（`print8x10`）
- 背景:
  - `grid`: 白
  - `filmStrip`: 黒
- 仕上げ調整: すべてニュートラル（OFF）
- 撮影日キャプション自動表示: OFF（フィルム風のシンプルな見た目を既定で維持する。オーナー判断 2026-07-17）

## 役割分離

- 見た目パネル: 余白・間隔・背景など、レイアウトの設定
- 調整パネル: シート全体の仕上げ。**「グループアイコン行 → 設定」の二階層**
  （色: モノクロ・色温度 / 階調: コントラスト・フェード・周辺減光 / 質感: 粒状感 / 刻印: デート焼き込み。
  今後 項目が増えてもグループ追加で収める。オーナー指示 2026-07-18）。
  写真ごとのレタッチは提供しない。粒状感・周辺減光はネガ / レンズ由来なので写真単位に適用される
- 背景: 白 / 黒 / グレー / **バライタ**（温白 + ごく薄い周辺減光）/ **ライトテーブル**
  （発光ビュアー。フィルム刻印がアンバー発光・スプロケットが光の抜けになる）/ カスタム
- タイトルパネル: タイトル・サブタイトル・撮影日自動表示トグル
- 書き出しパネル（画像）: 用紙フォーマット・画質（印刷系用紙のときのみ）
- 書き出しパネル（動画）: 速度・表示行数・全体表示

## 書き出し画質（実寸プリント対応, 2026-07-18）

- 画像書き出しパネルに品質選択（3 段階）を追加: `画面向け（現行）` / `プリント（標準）` / `プリント（高精細）`
  （既存の速度ピッカー `ゆっくり`/`ふつう`/`はやく` と同じ命名方針。ラベルに DPI 数値は出さない。
  オーナー指摘: 「Issue の説明では DPI 指定にしていたが、デジタルツールであるのでピクセルまたは
  定性的な表現にするのがいい」。数値は用紙ごとに変わり一貫したラベルにならないため、
  既存の `displayName` 命名慣習（定性語のみ）を踏襲した）
- 内部的な対応関係: 標準品質 = 300dpi 相当、高品質 = 600dpi 相当（`ImageExportQuality.printDPI`）。
  DPI はユーザーに見せる文言ではなく印刷解像度を正しく算出するための内部実装詳細
- 印刷系用紙（8x10 / 4x6 / A4）を選んでいるときだけ品質選択を表示する。`flexible` / `story9x16`
  （物理的な印刷用紙を持たない画面向けフォーマット）のときは非表示にし、常に画面向け解像度になる
- 品質を明示的に選ばない場合の既定は `画面向け（現行）`（`ExportSheetUseCase.defaultPixelWidth` と同じ値。
  挙動の互換を壊さない）
- 安全上限 `ExportSheetUseCase.maxSafePixelWidth = 6000px`。計算結果がこれを超える場合は自動的に
  クランプする（メモリ使用量が過大にならないようにするため）。クランプ発生時のユーザー向け警告 UI は
  今回のスコープ外とし、将来の別 Issue で扱う

## 取り込みと並び順（2026-07-17）

- 取り込み時は撮影順に自動整列: EXIF 撮影日時 → なければファイル名の自然順
  （フィルムスキャンは EXIF が無く、ファイル名 = スキャン順 = 撮影順のことが多いため）
- 手動並べ替え済みのシートは崩さない（追加分だけ末尾に撮影順で追記）
- 全体の並べ替えは [+] メニューの「撮影順に並べ替え」で明示的に実行

## スタイル

- `グリッド`（インデックスプリント風）/ `フィルム`（ベタ焼き風）/ `スリーブ`（ネガファイル風。2026-07-17 追加）
- スリーブの既定背景は `paperGray`（アーカイブ台紙のイメージ）
- **スリーブの中身は実物どおり「切ったフィルムストリップ」**（FilmStripRow を半透明ポケットに収める。2026-07-18）。
  左端にバインダーのパンチ穴余白。フィルム系オプション（フィルム選択・縁の文字等）はフィルムスタイルと完全共有
- **フィルムは「手貼り」表現**（2026-07-18）: 行ごとに僅かな横オフセット（±0.4%）と回転（±0.22°）、
  ストリップ端に切り残し余白（leader 1.2%）。値は `SheetLayoutMath` の決定論的ハッシュ乱数
  （splitmix64）なので、プレビュー・書き出し・スナップショットで常に同じ絵になる（WYSIWYG 維持）

## フィルムフォーマット

- 35mm フルフレーム / ハーフ / 120（6×6・6×7）
- 120 はパーフォレーション（スプロケット穴）なし・コマ番号は「8A」併記なしで描く（実物準拠）
- ハーフは **2 コマ 1 組**（ペア内は密着・ペア間は広め。2 コマで 35mm 1 コマ分という実物準拠。2026-07-18）
- スプロケット穴はプリントでは「光の素通し = レベートよりさらに黒い」、ライトテーブルでは「光が抜けて明るい」
- 各コマに ±2% の露光ムラ（決定論ノイズ。ラボ焼きの揺らぎ）

## フィルム的ディテール（すべて default OFF。シンプルな見た目を既定で維持）

- デート焼き込み: クォーツデート風のオレンジ日付（`'26 7 17`）を各コマ右下に刻印。EXIF 撮影日がある写真のみ
- 縁のコマ番号刻印: エッジテキストに `▸12` を併記
- エッジテキストのプリセット: 実在フィルム銘柄は商標リスクがあるため汎用表記のみ（COLOR 400 / MONO 400 等）

## 動画 UX

- 進捗オーバーレイ表示（%）
- 書き出し処理は CVPixelBuffer 直接描画で高速化済み

## やらないこと（復活させない）

- **赤いマークで写真をセレクトした風に見せる機能**（オーナー判断 2026-07-17。
  参考にした Web サービスの模倣になりすぎるため採用しない）
- 写真ごとの個別レタッチ（調整はシート全体の仕上げに限定する）
